# 项目完成度说明（答辩用）

## 一、项目目标

本项目实现一个面向校园/园区场景的失物招领系统前端，覆盖普通用户和管理员两类角色，支持从信息发布到审核管理的完整流程。

## 二、当前完成度结论

- 综合完成度：约 80%+（可演示、可交付课程答辩）
- 功能完成度：高
- 工程化与质量：中上（已建立基础门禁与鉴权闭环）

## 三、已完成核心内容

1. 双角色业务闭环
   - 用户端：登录注册、失物/招领查看与发布、留言反馈、个人中心
   - 管理端：公告管理、信息审核、账号管理

2. 鉴权闭环
   - 已增加路由守卫，未登录访问受限页面会重定向
   - 按角色限制管理员与用户端页面访问

3. 接口层规范化
   - 建立统一请求实例与拦截器
   - 登录相关接口已从页面内联请求抽离到 `api` 层
   - 统一处理 token 注入和 401/403 失效

4. 代码质量提升
   - 已接入 ESLint 与 `npm run lint`
   - 已清理历史重复 key、无用参数、无效转义、调试日志等问题
   - 当前 lint 可通过，无阻塞错误

## 四、可展示亮点

- 角色权限控制完整，演示效果直观
- 管理员审核流程与用户发布流程形成闭环
- 工程化能力有体现（请求层、路由守卫、代码检查）

## 五、后续优化方向

1. 测试体系补强
   - 增加关键路径单元/组件测试（登录、权限跳转、列表查询）

2. 接口层全面抽离
   - 将剩余页面中的直接 `axios` 调用继续迁移至 `src/api`

3. 体验与稳定性优化
   - 统一异常提示文案
   - 提升表单校验一致性
   - 优化大列表分页交互与加载态

## 六、答辩一句话总结

本项目已具备完整业务流程和可演示交付能力，并在鉴权、工程化和代码质量方面完成了针对性完善，能够支撑课程答辩场景。

## 八、5分钟答辩口述稿

各位老师好，我汇报的项目是失物招领系统前端。项目定位是面向校园或园区，核心目标是把“用户发布信息 - 平台审核管理 - 用户沟通反馈”这一流程完整打通。

先说业务完成情况。系统分为用户端和管理员端：

用户端支持登录注册、失物信息发布、招领信息发布、信息查询、留言联系、反馈建议和个人中心管理。  
管理员端支持公告管理、失物/招领信息审核、用户与管理员账号管理。  
因此从使用流程上，项目已经具备可演示的闭环能力。

再说我这次重点完善的工程能力。

第一是鉴权与权限控制。我在路由层增加了全局守卫，受保护页面必须有登录态；同时按角色限制路由访问，管理员和普通用户只进入各自有权限的页面。这样可以避免未登录直达业务页，以及越权访问问题。

第二是请求层规范化。我新增了统一请求实例和拦截器，把 token 注入、异常拦截、401/403 失效处理集中管理。并且把登录相关接口从页面中的分散请求抽到 API 层，后续模块扩展时更容易维护。

第三是代码质量提升。项目已经接入 ESLint，新增了 `npm run lint` 检查流程。我对历史存量问题做了系统清理，包括重复字段定义、无用参数、无效转义、调试日志等，目前 lint 已经可以稳定通过，工程状态比初始版本更可控。

总体评价方面，我认为项目已经达到课程答辩交付标准。功能完整性较高，工程化和质量保障由“可运行”提升到了“可维护”的基础阶段，综合完成度可以给到 80% 以上。

最后说后续计划，主要有三点：  
第一，补关键路径测试，覆盖登录、鉴权跳转和核心列表查询；  
第二，继续把剩余页面中的直连请求迁移到统一 API 层；  
第三，进一步优化用户体验，比如异常提示统一、加载状态和分页交互细化。  

以上是我的汇报，谢谢老师。
